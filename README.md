# Engo (잉고)

영어 표현·문장·문법을 직접 정리하고, 정리한 것을 **바탕화면 메모지**로 다시 보는 윈도우 프로그램입니다.
서버 없이 이 컴퓨터에만 저장되고, 다른 기기와는 파일을 주고받아 합칩니다.

## 실행

**`Engo.bat` 을 더블클릭하면 끝입니다.** 필요한 것이 없으면 알아서 준비합니다.

처음 실행하면 설치 창이 먼저 뜹니다.

- **이미 이 컴퓨터에 있는 것**과 **없어서 새로 받아야 하는 것**을 나눠서 보여주고,
  받을 용량을 알려준 다음 **동의를 받고 나서야** 내려받습니다.
- 이미 설치된 라이브러리는 다시 받지 않고 그대로 씁니다.
- 받은 것은 전부 이 폴더 안 `.venv` 에만 들어갑니다. 시스템에 설치된 Python 은 건드리지 않습니다.
- 무엇을 새로 받았는지는 `install-manifest.json` 에 기록됩니다 — 나중에 지울 때
  "원래 있던 것"과 "받은 것"을 구분하는 근거입니다.

두 번째 실행부터는 창 없이 바로 시작합니다.

> Python 3.10 이상이 필요합니다. 없으면 설치 안내가 나옵니다.
> 폴더 경로가 너무 길면(Windows 260자 제한) 미리 알려주니 `C:\Engo` 처럼 짧은 곳으로 옮기세요.

직접 준비하고 싶다면:

```
py -m venv .venv
.venv\Scripts\python.exe -m pip install -r requirements.txt
```

읽어주기용 음성 파일은 프로그램을 처음 켤 때 받을지 물어봅니다 (기본 4종, 약 336MB).
명령줄에서 미리 받으려면 `.venv\Scripts\python.exe tools\get_voices.py` 를 실행하세요.

## 지울 때

데이터 탭 맨 아래 `프로그램 삭제`.

1. **정말 지울지** 한 번 묻습니다 — 학습 데이터는 되돌릴 수 없습니다.
2. **내려받은 구성 요소**(음성 파일, `.venv`)를 함께 지울지 **따로** 묻습니다.
   원래 이 컴퓨터에 있던 라이브러리는 목록에 "지우지 않습니다"로 표시되고 손대지 않습니다.
   Engo가 만들지 않은 것 — 직접 만든 `.venv`, 직접 넣은 음성 모델, 데이터 폴더 안의
   다른 파일 — 은 어떤 경우에도 지우지 않고 남긴 목록으로 보여줍니다.
3. 프로그램 폴더 자체는 남습니다. **직접 지우세요** (마지막 안내 창에서 폴더를 열어 줍니다).

실행 중인 `.venv` 는 Windows 가 잠그고 있어 바로 지울 수 없으므로, 프로그램이 완전히
닫힌 뒤에 자동으로 정리됩니다.

## 기능

| 탭 | 하는 일 |
|---|---|
| 1. 영어 표현 | 공부한 표현의 원문·한글 뜻·메모·출처·태그를 직접 정리 |
| 2. 원문 해석 | 영어 원문을 붙여넣으면 **문장마다 한 행**으로 나뉘고, 행마다 `내 해석` 칸과 `피드백 메모` 칸에 직접 입력 |
| 3. 외우고 싶은 문장 | 특히 외우고 싶은 문장을 원문·번역으로 정리 (⭐ 우선 표시) |
| 4. 문법 | `주요 표현` 칸과 `설명` 칸을 나눠서 정리 (예문 칸도 있음) |
| 5. 데이터 | 내보내기 / 가져오기(병합) / CSV / 백업 |

### 복습 메모지

트레이 아이콘 우클릭 → `📌 복습 메모지`, 또는 메인 창에서 `Ctrl+N`.

- 창틀 없는 메모지가 뜨고, **제목 줄을 잡아 끌면** 이동, 오른쪽 아래 모서리로 크기 조절
- 기본은 **한글 뜻을 가린 상태** — 항목을 클릭하면 뜻이 열리고 `알아요` / `헷갈려요` 버튼이 나옵니다
- `알아요`를 누르면 복습 단계가 오르고 목록에서 빠지고, `헷갈려요`는 0단계로 돌아갑니다
- 위치·크기·색·투명도는 기억되고, 프로그램을 껐다 켜도 **띄워뒀던 메모지가 그대로 복원**됩니다
- 우클릭 메뉴에서 `항상 위에 두기`, 색상 변경 가능
- 범위는 `오늘 공부한 것` / `헷갈리는 것만` / 특정 날짜 / 특정 태그 중에 고릅니다

메모지는 한 번에 30건씩 보여줍니다. 다 풀고 `⟳`를 누르면 다음 묶음이 나옵니다.

### 윈도우 시작할 때 자동 실행

트레이 아이콘 우클릭 → `윈도우 시작할 때 자동 실행`. 부팅하면 트레이에 상주하면서
전에 띄워뒀던 메모지를 다시 띄웁니다.

### 색 테마 바꾸기

메뉴 `보기 → 색 테마`, 또는 트레이 아이콘 우클릭 → `색 테마`.
바이올렛 / 오션 / 포레스트 / 선셋 / 미드나잇(어두운 테마) 다섯 가지이며,
메인 창과 복습 메모지가 함께 바뀝니다. 고른 테마는 기억됩니다.

메모지 색은 테마의 파스텔 6색에서 고릅니다 — 테마를 바꾸면 메모지 색조도 따라갑니다.

### 한/영 전환

메뉴 `보기 → 언어 / Language`, 또는 트레이 메뉴에서.
탭 이름·버튼·안내 문구·메뉴·상태바까지 UI 전체가 바뀝니다.
공부한 내용(표현·문장·문법)은 당연히 그대로입니다.

### 읽어주기 (TTS)

인터넷 없이 이 컴퓨터에서만 돌아갑니다 (Piper 신경망 음성).

- 표현·문장·문법 탭의 🔊 — 영어를 읽어줍니다 (문법 탭은 예문)
- 원문 해석 탭의 🔊 — 고른 문장, 선택이 없으면 지문 전체
- 복습 메모지 — 항목마다 🔊. 뜻을 열기 전에 소리부터 들을 수 있습니다
- 단축키 `Ctrl+P`

`보기 → 읽어주기 음성`에서 고릅니다. **칸이 4개**이고, 네 개 모두 첫 실행 때 받습니다.

| 칸 | 기본 음성 | 음정 | 크기 | 합성 속도 |
|---|---|---|---|---|
| 1 | 여성 · 기본 (hfc_female) | 251 Hz | 60 MB | 5.5초 분량에 0.30초 |
| 2 | 남성 · 기본 (hfc_male) | 167 Hz | 60 MB | 0.24초 |
| 3 | 여성 · 낮은 톤 (Lessac) | 187 Hz | 109 MB | 0.99초 |
| 4 | 남성 · 낮은 톤 (Ryan) | 154 Hz | 115 MB | 0.93초 |

`낮은 톤` 쪽이 더 큰 모델(`high` 등급)이지만 **이름 그대로 더 낮게 말합니다.**
크다고 더 자연스럽게 들리지는 않아서 기본값은 위쪽 두 개입니다.

### 음성 칸 직접 설정

`데이터` 탭 → `읽어주기 음성` → **음성 설정**.

4칸 각각에 **원하는 음성 파일과 이름**을 정할 수 있습니다. 여기서 붙인 이름이 읽어주기
메뉴에 그대로 나옵니다. 목록에는 미국·영국 음성 22종이 들어 있고, 아직 받지 않은
음성은 `없음`으로 표시되며 `받기`로 그 자리에서 내려받습니다.
`기본값으로` 를 누르면 위 표의 네 가지로 돌아갑니다.

### 물음표·느낌표

네 음성 모두 **`?`에는 끝을 올려 읽습니다.** 같은 문장을 마침표와 물음표로 합성해
문장 끝 음정을 재본 결과입니다.

| 음성 | `.` 로 끝날 때 | `?` 로 끝날 때 | 차이 |
|---|---|---|---|
| 여성 · 기본 | 162 Hz | 335 Hz | **+173** |
| 남성 · 기본 | 73 Hz | 188 Hz | **+115** |
| 여성 · 낮은 톤 | 105 Hz | 244 Hz | **+139** |
| 남성 · 낮은 톤 | 90 Hz | 153 Hz | **+63** |

반면 **`!`는 마침표와 거의 같게 읽습니다** (끝 음정 차이 2~7 Hz). 느낌표를 살려주는
모델은 없으니, 강조가 필요하면 문장을 짧게 끊는 편이 낫습니다.

### 끝까지 읽기

Ryan 음성은 모델이 **끝에 무음을 전혀 남기지 않아** 마지막 소리가 잘려 나갔습니다
(다른 음성은 15~170 ms의 여유가 있습니다). 그래서 합성할 때 스트리밍 API로 모든
조각을 받아 쓰고 끝에 250 ms의 무음을 붙입니다. 네 음성 × 세 종류 문장 모두
무음으로 끝나는지 테스트가 확인합니다.

음성을 바꾸면 상태 표시줄 오른쪽에 `🔊 음성 바꾸는 중…`이 뜹니다.
모델은 처음 쓸 때 올라오고 **2분간 쓰지 않으면 해제**되며, 합성은 별도 스레드에서
돌아 창이 멈추지 않습니다.

### 음성 파일 받기

음성 파일은 저장소에 들어 있지 않습니다. **프로그램을 처음 켜면 기본 4종을 받을지 물어보고**,
`나중에`를 골랐더라도 `데이터` 탭의 **읽어주기 음성**에서 언제든 받을 수 있습니다.
받지 않아도 나머지 기능은 그대로이고 🔊 버튼만 숨겨집니다.

받는 도중 취소해도 조각 파일이 남지 않습니다 — 파일은 `.part`로 받은 뒤
완료됐을 때만 제자리로 옮겨지므로, 끊긴 모델이 로드되어 프로그램이 죽는 일은 없습니다.

## 업데이트

`데이터` 탭 맨 위의 **프로그램 업데이트**에서 확인하고 설치합니다.
탭을 열 때 자동으로 한 번 확인하고(프로그램 시작 시에는 확인하지 않습니다),
새 버전이 있으면 탭 이름에 `● ` 표시가 붙습니다.

GitHub에 새 릴리스가 올라오면 **`업데이트 설치` 버튼 하나로 받아집니다.**
git이 없어도 됩니다 — 릴리스 압축파일을 받아 프로그램 폴더에 풀어넣습니다.

- 압축파일이 Engo 구조인지 확인한 뒤에만 파일을 바꿉니다
- 설치는 같은 볼륨 안에서 통째로 바꿔치기(rename)라 빠르고, 도중에 실패하면
  **자동으로 원래대로 되돌립니다** (이전 파일은 `.update-backup`에 남습니다)
- 릴리스에서 삭제된 파일도 함께 사라집니다 — 옛 파일이 남아 새 코드를 가리는 일 없음
- `voices/`, `.venv/`, 학습 데이터는 절대 건드리지 않습니다
- 폴더의 파일을 직접 고쳤다면 업데이트를 멈춥니다 (고친 내용 보호)

설치 후 프로그램을 껐다 켜면 적용됩니다.

### 새 버전 내보내기 (개발용)

업데이트 확인은 **커밋이 아니라 릴리스**를 봅니다. main 에 푸시만 하면 다른 기기는
계속 "최신"이라고 나옵니다 — 푸시는 배포가 아닙니다.

```
python tools/release.py           # 1.0.1 → 1.0.2
python tools/release.py 1.1.0     # 버전 직접 지정
python tools/release.py --dry-run # 무엇을 할지만 보기
```

버전 올리기 · 커밋 · 태그 · 푸시 · 릴리스 생성을 한 번에 하고, 마지막에
`releases/latest` 를 다시 읽어 **이전 버전 사용자에게 실제로 보이는지 확인**합니다.

## 다른 기기와 합치기

서버가 없으므로 **파일로** 주고받습니다. USB든 클라우드 드라이브든 상관없습니다.

1. A 컴퓨터: `데이터` 탭 → `파일로 내보내기` → `.seb` 파일이 생깁니다
2. 파일을 B 컴퓨터로 옮깁니다
3. B 컴퓨터: `데이터` 탭 → `파일 선택…` → `병합 미리보기`로 확인 → `합치기`

반대 방향도 똑같이 하면 양쪽이 같은 상태가 됩니다.

### 합칠 때 규칙

- 같은 항목이 양쪽에 있으면 **더 나중에 수정한 쪽**이 남습니다
- 한쪽에만 있는 항목은 그대로 추가됩니다
- 한쪽에서 지운 항목은 다른 쪽에서도 지워집니다
- **같은 파일을 두 번 합쳐도 결과는 같고**, 합치는 순서를 바꿔도 결과는 같습니다
- 지문(원문 해석)은 **문장 행 단위로** 합쳐집니다. A에서 1번 문장을, B에서 2번 문장을
  해석했다면 합친 뒤 둘 다 남습니다

`마지막 내보내기 이후 변경분만` 으로 내보내면 파일이 작아집니다. 결과는 전체 내보내기와 같습니다.

### 그 밖에

- **CSV**: 엑셀에서 정리한 표를 가져오거나, 정리한 내용을 엑셀로 내보냅니다.
  (CSV에는 항목 고유번호가 없어서 **병합이 아니라 추가**입니다)
- **백업**: 합치기 전에 자동으로 백업되며, 최근 10개까지 보관합니다
- **정리**: 180일 넘은 삭제 기록을 지우고 파일 크기를 줄입니다

`ENGO_HOME` 환경변수로 위치를 바꿀 수 있습니다.

## 메모리에 대해

메모지를 하루 종일 띄워두는 프로그램이라 다음을 실측하며 만들었습니다.
(3,000건이 든 데이터베이스 기준)

| 상태 | 비용 |
|---|---|
| 메모지 1장 추가 | **약 2.6 MB** (같은 프로세스 안의 창이라 프로세스가 늘지 않음) |
| 메인 창 열기 | 약 9 MB, 22 ms |
| 메인 창 닫기 | 위젯 322개 전부 해제 — 트레이와 메모지만 남음 |
| 3,000행 목록 | 처음엔 200행만 읽고, 스크롤하는 만큼만 더 읽음 |

- 목록은 화면에 보이는 만큼만 데이터베이스에서 읽습니다 (`Qt`의 `fetchMore`)
- 긴 메모는 목록 조회 단계에서 잘라서 가져오고, 전체 내용은 항목을 선택할 때만 읽습니다
- 탭은 처음 열 때 만들어집니다 — 안 쓰는 탭은 메모리를 차지하지 않습니다
- 메인 창을 닫으면 창을 **없앱니다**. 트레이 아이콘이 프로그램을 붙잡고 있고, 메모지는 그대로 남습니다

```
.venv\Scripts\python.exe tests\test_setup.py     설치 판단·삭제 대상 (내려받지 않음)
.venv\Scripts\python.exe tests\test_first_run.py --real   빈 폴더에서 실제 설치 (약 325MB)
```

`test_render.py`는 실제 화면이 필요합니다. 컨트롤이 배치되고 `visible=True` 인데도
아무것도 안 그려지는 경우가 실제로 여러 번 있었기 때문에, 칠해진 픽셀을 직접 셉니다.
버튼이 강조색으로 채워지는지, 다크 테마에서 글자 대비가 충분한지, 스크롤바가 둥근
모서리를 침범하지 않는지, 탭을 옮길 때 엉뚱한 창이 뜨지 않는지도 함께 봅니다.
